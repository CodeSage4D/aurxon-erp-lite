FEATURES ADDED
--------------------------------------------------------------------------------
1. Founder Support Access Impersonation Session
   - Implemented dynamic, time-limited, and audited support tokens.
   - Allows Founders/Super Admins to troubleshoot organization tenant issues
     with customer permission, while enforcing a privacy boundary preventing
     unauthorized direct access to sensitive student, staff, and financial data.
2. Context Switching Context Resolution
   - Enhanced API context switches to return branding assets (logo URLs,
     branding primary colors, active pack configuration) in the JWT payload.
3. Multi-Tenant Registration & Onboarding Lifecycle Workflow
   - Implemented registration approval, rejection, and activation workflows.
   - Restricts login activation until a trial key is verified.

FEATURES IMPROVED
--------------------------------------------------------------------------------
1. Overhaul Workspace Header UI
   - Upgraded to an enterprise SaaS design displaying the active organization
     logo, institution name, active workspace role, active packages,
     notification indicators, and a clean profile menu.
2. Visual Standardization (De-gradient Transformation)
   - Removed hot/warm pink/red "Instagram-style" gradients.
   - Replaced UI background panels with professional, high-contrast slate,
     indigo, and sky blue enterprise themes.
3. Authentication Experience (Zero-Flash Shield)
   - Created a client-side state machine in page.tsx that blocks background page
     queries and visual flicker on reload.
   - Shows a premium full-screen loading skeleton during auth context switches.

BUGS FIXED
--------------------------------------------------------------------------------
1. Reports Dashboard Console Fetch Crash
   - Patched `getClassPerformanceReportApi` and ReportsDashboard.tsx to guard against
     empty or undefined `examId` queries.
   - Short-circuits invalid API calls directly in the client, eliminating
     CORS block errors on NestJS 404 pages.
2. Route Permissions & Role Leakages
   - Hardened RBAC routes to block cross-role access (e.g., standard users or
     finance managers trying to access backup management routes).
