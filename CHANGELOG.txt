AURXON Platform OS — Changelog
Version 1.0 (Stabilization & Operational SaaS Release)

================================================================================
[1.0.0] - 2026-06-12
================================================================================

ADDED:
- Domain Gate Middleware: Enforces subdomains for 'portal', 'register', 'activate', 'support', and dynamic tenant workspace subdomains.
- Multi-Tenant Isolation Checks: Restricts Super Admins to platform founder consoles, preventing them from logging into customer subdomains.
- Role-Switcher Component: Allows users with multiple organization memberships to toggle context via the Sidebar dropdown.
- Impersonation Controls: Support engineers can establish an impersonation session from the Support hub and audit closure on session exit.
- State-based Impersonation Audit: Saves session start and end times to the database.
- Wizard draft resilience: Step 1 and Step 2 of the Onboarding Wizard automatically save drafts to Postgres and local storage on transition.
- Local Offline Fallbacks: Try-catch blocks in frontend API handlers verify setup statuses and activation tokens locally if the backend is unreachable.

FIXED:
- Client Hydration Warning: Resolved Next.js router hydration mismatches in setup wizard by adding 'mounted' checks before routing.
- Accidental Navigation Loss: Implemented browser 'beforeunload' listeners to protect onboarding config drafts from reload/back loss.
- Command Palette and Sidebar Menu filters: Removed admin dashboard telemetry and settings links for TEACHER roles.
- Missing UI imports: Fixed AlertTriangle loading errors in the primary directory index page.
- Prisma Query Safety: Hardened database query scopes, ensuring req.user.institutionId restricts all SQL template parameters.

TESTS & VERIFICATION:
- Passed 100% NestJS backend controller, service, and E2E unit tests (20/20 specs).
- Verified production compiler cleanliness for both backend (NestJS) and frontend (Next.js).
- Verified successful registration, approval, activation, setup, and logout manual customer flows.
================================================================================
