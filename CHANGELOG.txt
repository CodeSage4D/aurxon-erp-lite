AURXON Platform OS — Changelog
Version 1.1 (Domain Re-Architecture & Final Stabilization)

================================================================================
[1.1.0] - 2026-06-12
================================================================================

ADDED:
- Domain Schema Models: Added Section and Enrollment tables to replace mutable grouping logic with immutable historical tracking in School ERP.
- Hospital ERP structure: Added HospitalDepartment, MedicalUnit, Ward, Bed, and PatientEpisode models.
- Corporate ERP structure: Added CorporateDivision, CorporateDepartment, CorporateTeam, and WorkAssignment models.
- Onboarding Setup Reset API: Added Post `/founder/institutions/:id/reset-wizard` to allow platform owners to override and reset onboarding wizard progress in Postgres.
- Setup Reset UI Action: Integrated "Reset Setup" button inside the organization detail cards in the Founder Console.
- Section API Routes: Implemented endpoints to query and create Sections in Class structures.

MODIFIED:
- Student Registration: Admitting a student automatically generates a Class Section A and creates an initial Enrollment history record.
- Student Promotion: Promoting a student registers a new Enrollment history record rather than overwriting historical Class data.
- Student view detail payload: Student findOne API returns historic enrollment arrays with their respective class, section, and academic year details.

FIXED:
- Local database migration sync successfully applied to Neon master database instance.
- Verified Next.js Turbopack and NestJS builds compile without warnings.
- Frontend AuthStage layout gate verified to prevent dashboard UI loading flashes.
================================================================================
